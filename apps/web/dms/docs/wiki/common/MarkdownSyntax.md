# 마크다운(Markdown) 문법 참고표

## 1. 기본 서식

### 제목 (Headers)
```
# 제목 1
## 제목 2
### 제목 3
#### 제목 4
##### 제목 5
###### 제목 6
```

### 텍스트 강조
```
*기울임체* 또는 _기울임체_
**굵게** 또는 __굵게__
***굵은 기울임체*** 또는 ___굵은 기울임체___
~~취소선~~
```
*기울임체* / **굵게** / ***굵은 기울임체*** / ~~취소선~~

## 2. 목록

### 순서 없는 목록
```
- 항목 1
* 항목 2
+ 항목 3
```
- 항목 1
* 항목 2
+ 항목 3

### 순서 있는 목록
```
1. 첫 번째 항목
2. 두 번째 항목
3. 세 번째 항목
```

### 중첩 목록
```
- 항목 1
  - 하위 항목 A
    - 더 깊은 하위 항목 i
- 항목 2
```

## 3. 링크 및 이미지

### 링크
```
[링크 텍스트](https://www.example.com)
[제목 있는 링크](https://www.example.com "링크 제목")
<https://www.example.com> (자동 링크)
```

### 이미지
```
![대체 텍스트](이미지URL)
![대체 텍스트](이미지URL "이미지 제목")
```

## 8. 이스케이프 문자
```
\* 별표를 문자 그대로 표시
\_ 언더스코어를 문자 그대로 표시
\` 백틱을 문자 그대로 표시
\# 해시 기호를 문자 그대로 표시
```
\* 별표를 문자 그대로 표시

## 15. 주의사항 및 팁

- 문단 구분은 빈 줄로 합니다.
- 문자 그대로 표시하려면 `\` 기호를 사용합니다.
- 대부분의 마크다운 에디터는 실시간 미리보기를 제공합니다.
- GitHub, GitLab, Notion 등 플랫폼마다 지원하는 문법에 약간의 차이가 있을 수 있습니다.
- 복잡한 표는 [Tables Generator](https://www.tablesgenerator.com/markdown_tables)와 같은 도구를 활용하세요.
- VS Code에서는 "Markdown All in One" 확장 프로그램을 사용하면 편리합니다.

## 16. 색상 사용 (확장 문법 - 일부 플랫폼만 지원)

```
<font color="red">빨간색</font>
```

```
  펼쳤을 때 표시될 내용
</details>
```
