# 문서 구조 정리 계획 (DMS)

> 작성일: 2026-01-21  
> 상태: 계획 수립  
> 목적: DMS 문서를 PMS 구조와 대칭으로 정리하기 위한 기준을 확정

---

## 목표 구조

```
docs/
  dms/               # 도큐먼트 관리 시스템 문서
  dms/common/        # DMS 공통 가이드/프로세스/운영
  dms/api/           # DMS API 문서 (추후)
  dms/architecture/  # DMS 아키텍처
  dms/database/      # DMS DB 설계/스키마 (추후)
```

---

## 이관 원칙

1. **DMS 문서는 `docs/dms`로 수렴**
2. **PMS와 동일한 구조를 유지**하되, 내용은 독립적으로 작성
3. 공통 문서가 필요할 경우 **PMS/DMS에 동일 문서를 복제**하여 관리

---

## 실행 단계

1. DMS 소스 반영 후 문서 스캔
2. PMS 구조와 동일한 디렉토리 생성
3. 문서 이동 + 링크/인덱스 갱신
4. 체크포인트 커밋
