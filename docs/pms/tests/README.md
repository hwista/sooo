# 테스트 문서

## 1. 개요

이 폴더는 SSOO 시스템의 기능별 테스트 케이스를 정리합니다.

- **액션 문서**: 핵심 테스트 케이스 요약 (Quick Check)
- **테스트 문서**: 상세 시나리오, 엣지 케이스, 자동화 대상

---

## 2. 문서 구조

```
docs/pms/tests/
├── README.md                     ← 현재 문서
├── auth/                         ← 인증 관련 테스트
│   ├── login.md                  ← 로그인 테스트
│   ├── logout.md                 ← 로그아웃 테스트
│   └── refresh_token.md          ← 토큰 갱신 테스트
└── project/                      ← 프로젝트 관련 테스트 (작성 예정)
```

---

## 3. 테스트 케이스 형식

각 테스트 케이스는 다음 형식을 따릅니다:

```markdown
### TC-XX: 테스트 케이스 제목

| 항목 | 내용 |
|------|------|
| **우선순위** | P0 (Critical) / P1 (High) / P2 (Medium) / P3 (Low) |
| **전제조건** | 테스트 실행 전 필요한 상태 |
| **입력** | 테스트 입력 값 |
| **실행 단계** | 1. 단계1 → 2. 단계2 → ... |
| **예상 결과** | 기대하는 출력/상태 변화 |
| **검증 항목** | 확인해야 할 세부 사항 |
| **자동화** | ✅ 가능 / ❌ 수동 필요 |
```

---

## 4. 우선순위 정의

| 등급 | 설명 | 예시 |
|------|------|------|
| **P0** | 핵심 기능, 반드시 통과해야 함 | 로그인, 로그아웃 |
| **P1** | 주요 기능, 릴리즈 전 필수 확인 | 토큰 갱신, 권한 검증 |
| **P2** | 보조 기능, 가능하면 확인 | 에러 메시지 형식 |
| **P3** | 엣지 케이스, 시간 여유 시 확인 | 동시 로그인 처리 |

## 6. 자동화 계획

### Phase 1: API 테스트 (Jest + Supertest)
```bash
# 서버 테스트
cd apps/server
pnpm test
pnpm test:e2e
```

### Phase 2: E2E 테스트 (Playwright)
```bash
# 웹 E2E 테스트
cd apps/web-pms
pnpm test:e2e
```

### Phase 3: CI/CD 통합
- GitHub Actions에서 PR마다 자동 테스트
- 테스트 커버리지 리포트
