# 프로젝트 라이프사이클 (Project Lifecycle)

> 최종 업데이트: 2026-01-20

프로젝트의 전체 생애주기를 정의합니다.

---

## 전체 흐름

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           OPPORTUNITY (기회)                            │
├──────────────┬──────────────┬──────────────────────────────────────────┤
│   waiting    │  in_progress │              done                        │
│   (대기)     │   (진행중)   │   won(수주) / lost(실주) / hold(보류)    │
└──────────────┴──────────────┴──────────────────────────────────────────┘
                                    │
                                    │ won → execution 전환
                                    ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                           EXECUTION (실행)                              │
├──────────────┬──────────────┬──────────────────────────────────────────┤
│   waiting    │  in_progress │              done                        │
│   (대기)     │   (진행중)   │            (완료)                        │
└──────────────┴──────────────┴──────────────────────────────────────────┘
                                    │
                                    │ 운영 전환 여부 선택
                                    ▼
                    ┌───────────────┴───────────────┐
                    │                               │
                    ▼                               ▼
              [SM 이관]                        [종료]
              handoff                         프로젝트 완료
```

---

## 1. Opportunity (기회) 단계

### 1.1 영업 베이스 프로젝트

1. **영업**: 제안/기회 등록 → `opportunity + waiting`
2. **영업**: 기회 관리 (필요 시 AM/PM 참여) → `opportunity + in_progress`
3. **영업/AM**: 계약 체결
4. **기회 종료**: `opportunity + done + (won/lost/hold)`
5. **수주 확정 시**: `execution + waiting` 으로 전환

### 1.2 고객 요청 베이스 프로젝트

1. **고객**: 요청
2. **AM**: 요청 확인 후 opportunity로 관리 (검토/딜/제안)
3. **AM**: 계약
4. 이후 프로세스는 영업 베이스와 동일

---

## 2. Execution (실행) 단계

1. **PM**: 실행 전환 및 핸드오프 수령 → `execution + waiting`
2. **PM**: 프로젝트 실행 → `execution + in_progress`
3. **PM**: 실행 중 중도금/중간보고 필요 시 → AM에 핸드오프
4. **AM**: 중도금 등 계약 이행
5. **PM**: 프로젝트 종료 → `execution + done`
6. **AM**: 최종 잔금 및 계약 관계 마무리
7. **(조건부) SM**: 운영 전환 프로젝트의 경우 핸드오프 수령 후 운영

---

## 3. Handoff (핸드오프)

역할 간 인계는 별도 트랙으로 관리됩니다.

| 타입 | 설명 |
|------|------|
| `PRE_TO_PM` | 기회 → PM (실행 인수) |
| `PRE_TO_CONTRACT_OWNER` | 기회 → 계약담당 |
| `EXEC_TO_CONTRACT_OWNER` | 실행 중 계약이행 |
| `EXEC_TO_SM` | 실행 → SM (운영 전환) |

---

## 4. Close (종료)

프로젝트 종료 시 검증 사항:

1. **종료조건 체크리스트** 완료 여부
2. **산출물 제출 상태** 확인 (confirmed)
3. **운영 전환 여부** 선택
   - 전환 → SM 핸드오프
   - 미전환 → 프로젝트 완료
